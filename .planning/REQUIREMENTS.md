# Requirements — KEX GROUP Dashboard

> Источник: подписанное ТЗ v1.5 + согласованные ответы

## Functional Requirements

### Auth
- [ ] OTP-авторизация по номеру телефона (Telegram-бот — основной, Mobizon SMS — fallback)
- [ ] JWT токены (access + refresh)
- [ ] 3 роли: OWNER, FIN_DIRECTOR, OPS_DIRECTOR
- [ ] Блокировка после 5 неудачных попыток на 15 минут (Redis)
- [ ] **Биометрия (Face ID / Touch ID / отпечаток пальца):**
  - [ ] Проверка доступности биометрии на устройстве (hasHardwareAsync + isEnrolledAsync)
  - [ ] Предложение включить после первого OTP-входа
  - [ ] Быстрый повторный вход: биометрия → refresh token → новый JWT
  - [ ] Настройка вкл/выкл в профиле пользователя
  - [ ] Fallback на OTP если биометрия недоступна или отключена
  - [ ] Backend: поле biometricEnabled в User, endpoint для включения/валидации
- [ ] Автовыход при неактивности (настраиваемый таймаут)
- [ ] AuditLog: запись каждого логина (userId, action, IP, timestamp)
- [ ] Реальная запись/чтение пользователя из PostgreSQL

### Drill-down навигация (4 уровня)

#### Уровень 1 — Компания (главный экран)
- [ ] Выручка по компании за период — одна большая цифра
- [ ] Расходы по компании за период
- [ ] Баланс = Выручка − Расходы
- [ ] Плитки по брендам (Burger na Abaya, Doner na Abaya) с выручкой и динамикой (+/-%)
- [ ] Раскрытие бренда → список точек с выручкой
- [ ] Переключатель периода: Сегодня / Неделя / Месяц / Прошлый месяц / Свой диапазон
- [ ] Время последней синхронизации

#### Уровень 2 — Точка
- [ ] Выручка точки по типам оплат (Наличные, Kaspi QR, Halyk QR, Яндекс)
- [ ] Расходы по группам статей ДДС (сумма по каждой группе)
- [ ] Распределённые затраты (доля ГО и Цеха по удельному весу)
- [ ] Финрезультат = Выручка − Прямые расходы − Распределённые затраты
- [ ] Недостачи и излишки по кассе
- [ ] График выручки по дням за период

#### Уровень 3 — Статьи (👑 Владелец + 📊 ФинДир)
- [ ] Статьи ДДС внутри группы с суммами
- [ ] Доля статьи в общих расходах точки (%)
- [ ] Сравнение с предыдущим аналогичным периодом (+/-)
- [ ] Пометка: прямая (iiko) или распределённая (1С)

#### Уровень 4 — Операции (только 👑 Владелец)
- [ ] Дата и время операции
- [ ] Сумма операции
- [ ] Комментарий / описание
- [ ] Источник: iiko (прямая) или 1С (расчётная)
- [ ] Коэффициент распределения (для распределённых)

### Распределение затрат по удельному весу
- [ ] Формула: Доля точки = Выручка точки ÷ Общая выручка всех точек (за тот же период)
- [ ] Распределяются: аренда ГО, ЗП операционистов, IT ГО, затраты Цеха, статьи ДДС без точки
- [ ] НЕ распределяются: прямые расходы точки (iiko), отгрузки Цеха (прямая привязка)
- [ ] Пересчёт при каждой смене периода

### Цех (производственное подразделение)
- [ ] Закупки сырья (бакалея, жир, напитки, сиропы)
- [ ] Отгрузки Цеха по точкам (прямая привязка)
- [ ] Поступления на счёт Цеха
- [ ] Затраты Цеха (аренда, ЗП, водители, оборудование, налоги)

### Отчёты
- [ ] ДДС сводный по всем точкам
- [ ] Затраты компании (ГО + Цех из 1С)
- [ ] Закупки и отгрузки Цеха
- [ ] Аналитика и тренды (графики)

### Синхронизация данных
- [ ] Автоматическое обновление из iiko каждые 15-30 минут (cron)
- [ ] Автоматическое обновление из 1С каждые 15-30 минут
- [ ] Лог синхронизации в SyncLog (SUCCESS/ERROR)
- [ ] Обработка недоступности API (retry, таймауты, circuit breaker)
- [ ] Pull-to-refresh (ручное обновление)

### Push-уведомления
- [ ] Firebase Cloud Messaging (бесплатно)
- [ ] Синхронизация не работает > 1 часа → push админу
- [ ] Выручка точки < 70% от среднего → push Владельцу + ОперДиректору
- [ ] Крупный расход > X ₸ → push Владельцу + ФинДиректору
- [ ] Настройки уведомлений в профиле

## Non-Functional Requirements

- [ ] iOS 14+ / Android 8.0+ (React Native + Expo)
- [ ] Главный экран: загрузка < 2 секунд
- [ ] Переход к точке: < 1 секунды
- [ ] Стабильная работа на 4G
- [ ] HTTPS для всех запросов (Nginx + Let's Encrypt)
- [ ] Фильтрация по роли на бэкенде (не только UI)
- [ ] Sentry: crash tracking (backend + mobile)
- [ ] Логирование через Winston/Pino
- [ ] Health check `/health` в каждом сервисе
- [ ] Prisma миграции (`migrate`, не `push`)
- [ ] Автоматические бэкапы PostgreSQL (ежедневно)
- [ ] Docker для всех backend сервисов
- [ ] GitHub Actions CI (lint + test + build)
- [ ] Swagger документация для API Gateway
- [ ] Локализация: русский + казахский
- [ ] Валюта: KZT (₸)
- [ ] Часовой пояс: Asia/Almaty

## Constraints

- Read-only приложение (данные вводятся в iiko и 1С)
- Единая PostgreSQL с multi-schema (auth + finance)
- HTTP/REST между сервисами (не gRPC, не очереди)
- Только телефон (без планшетной адаптации)
- Нет веб-версии (исключено из scope)
- Нет админ-панели (управление через Prisma Studio / SQL)
- Нет экспорта PDF/Excel
- Нет тёмной темы
- Секреты хранятся в `.env` (Vault — позже)
- Staging сервер — после первого релиза
