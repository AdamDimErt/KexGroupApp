## Decisions Made

- `resolveGroupCode` checks 'заработн' in addition to 'зарплат' to match both 'Заработная плата' and 'Зарплата' account name forms
- `syncDdsArticles` tries POST /v2/entities/list first (iiko v2 pattern), falls back to GET on failure for compatibility
- `ddsArticle.upsert` uses `groupId_code` compound unique key — `code` alone is not a unique constraint in DdsArticle schema
- `syncDdsTransactions` uses `findFirst({ where: { code, source: 'IIKO' } })` not `findUnique` — DdsArticle.code is optional and only unique per groupId
- Per-restaurant errors in `syncDdsTransactions` are caught and warned rather than propagated — consistent with `syncRevenue`/`syncExpenses` per-restaurant error handling patterns
