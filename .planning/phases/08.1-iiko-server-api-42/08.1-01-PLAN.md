    2. **async syncDdsArticles(): Promise<void>** — 
       - Get tenantId from first Restaurant record (or process.env.TENANT_ID)
       - Try GET first: makeRequest('/v2/entities/list?rootType=Account')
       - Parse XML response (XMLParser) — look for array of account items
       - If GET fails with 405, try POST: makePostJsonRequest('/v2/entities/list', { rootType: 'Account' })
       - For each account: resolve groupCode via resolveGroupCode(account.name)
       - Find DdsArticleGroup by { tenantId, code: groupCode }
       - prisma.ddsArticle.upsert where: { code: account.id } (iiko UUID as code)
       - Update: name, isActive. Create: groupId, name, code, source='IIKO', allocationType='DIRECT', isActive=true
       - For accounts with no parent restaurant (HQ-level): set allocationType='DISTRIBUTED'
       - logSync + Sentry on error, following existing pattern exactly
       - Log raw response keys at debug level (for first-run observability per research)

    3. **async syncDdsTransactions(dateFrom: Date, dateTo: Date): Promise<void>** —
       - Get all restaurants from DB
       - For each restaurant with iikoId:
         - makePostJsonRequest('/v2/cashshifts/list', { organizationId: restaurant.iikoId, dateFrom: ISO, dateTo: ISO })
         - LOW CONFIDENCE on response schema — add logger.debug to log raw response keys on first call
         - Iterate cashShifts (or whatever the response array key is) -> payIns + payOuts
         - For each movement: syncId = `dds:${shiftId}:${movementId}`
         - Find DdsArticle by code = movement.accountId
         - If article not found: logger.warn + skip (do NOT throw)
         - prisma.expense.upsert where: { syncId }, create with articleId, restaurantId, date, amount, comment, source='IIKO'
       - logSync SUCCESS with count of upserted records
       - On error: logSync ERROR + Sentry capture, following existing pattern
