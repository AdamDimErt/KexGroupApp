    - Export `ACCESS_MATRIX` as a `Record<string, string[]>` constant with exact 6 entries from CONTEXT.md Decision 2 (keys WITHOUT method prefix, just paths):
      ```
      '/dashboard/article/:id/operations': ['OWNER']
      '/dashboard/reports/dds': ['OWNER', 'FINANCE_DIRECTOR']
      '/dashboard/reports/company-expenses': ['OWNER', 'FINANCE_DIRECTOR']

      '/dashboard/article/:groupId': ['OWNER', 'FINANCE_DIRECTOR']
      ```
    - `@Injectable()` class `DataAccessInterceptor implements NestInterceptor`
    - `intercept(context: ExecutionContext, next: CallHandler): Observable<unknown>` method
    - Read `request.path` (NOT `request.route.path` — route.path may be undefined during interceptor phase)
    - Read `request.headers['x-user-role']` as string
    - Call `this.matchPattern(path)` to find matching ACCESS_MATRIX key
    - If match found AND (no role OR role not in allowed list): `throw new ForbiddenException('Access denied for role ${role ?? "unknown"}')`
    - If no match in matrix: passthrough (call `next.handle()`)
    - `private matchPattern(path: string): string | null` — iterate ACCESS_MATRIX keys, convert each to regex by replacing `:[^/]+` segments with `[^/]+`, anchor with `^...$`, test against path. Return first match or null.
    - IMPORTANT: Order the ACCESS_MATRIX keys so `/dashboard/article/:id/operations` is checked BEFORE `/dashboard/article/:groupId` — otherwise the shorter pattern matches first and blocks operations route from being recognized as OWNER-only.

    Imports: `Injectable, NestInterceptor, ExecutionContext, CallHandler, ForbiddenException` from `@nestjs/common`, `Observable` from `rxjs`.
